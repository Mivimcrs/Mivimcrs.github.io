怎样 0 成本打造自己的个人博客

发表于 2019-08-11 更新于 2020-11-19 分类于 教程

写博客是一种传统的互联网发声方式，博客也是博主的特殊个人空间，选择一个靠谱的平台尤其重要。QQ 空间、新浪博客和网易博客都是大众常用的博客平台，但这些平台有很多缺点：

  • 会员限制
  • 大量广告
  • 自定义难度高
  • 较少的言论自由

为了拥有一个完整控制权的个人博客，自己搭建是最好的选择。

但仅仅是为了写写文字，大多数人不可能为此去买一个服务器，又去学习编程。

「有没有几乎一键上手的方案啊！」

还真有一个简单又低成本的办法。

Hexo—— 一个快速、简洁且高效的博客框架，利用设置好的主题就能快速生产静态网页。

而且搭建超级简单，仅需做一些前置准备，后面就可以跟着教程复制代码回车就行。

最关键是主题很好看！

后期美化也比较容易，用户也多，遇到问题也容易解决。

废话少说，立马开干。

前置准备

一、安装 Node.js

使用 Hexo 需要安装 Node.js，所以我们第一步就是安装它。

  1. 点击我进入到 Node.js 官网；
  3. 安装过程一路默认就好；
  4. 安装结束后，利用 Win+R 快捷键打开运行
  5. 输入 CMD 打开命令提示符
  6. 在命令提示符窗口输入 node -v 和 npm -v，如果版本号出现的话，就代表安装成功了。

二 、注册 Github

我曾经因为搭建博客需要服务器托管从而妥协使用国内的博客平台，在不考虑商业作用下，花钱购买服务器，除非土豪，要不着实难以接受。

后来我发现可以通过 Github 的服务，达到服务器托管的效果，也因此搭建起了这个博客。

我们要使用 Github 的服务，肯定得拥有它家的账户。

如果你已经拥有了 Github 账户，即可跳过创建账户的步骤，从「建立仓库」的部分开始阅读就好。

如果你还没有账户，新建一个就好，具体教程：极客学院 github 注册教程。

三、建立托管博客的「仓库」

在 Github 是通过版本库来进行管理项目的，而这个版本库一般俗称为「仓库」。

  1. 打开 Github，这里默认你已经登录好账户了；

  3. 进入新建界面后，如图所示操作就好。

  4. 在「仓库」界面上方，找到 Settings 按钮，点进去。

  5. 在里面找到 GitHub Pages 部分，如下图操作。

    进到里面后。

  6. 在图上的位置可以看到这个「仓库」渲染后的静态网址，地址应该和你创建的仓库名相同。

搭建好博客后，这个网址就是你的博客网址。

安装 Hexo

根据下面清单再次检查自己的前置准备是否都完成了：

  • 安装 Node.js

  • 安装 Git

  • 注册 Github

  • 创建托管博客的 Github 仓库

  • 获得仓库静态网页地址

非常好，我们现在终于可以正式搭建自己的博客了。

「啥！现在才算开始搭建博客啊＼（≧Д≦）＜，我都快累死啦，那里算一步啊！」

淡定淡定，前面的步骤基本可以算是 60% 啦，其实搭建博客的步骤仅是全部行动的 20%。

一、给博客安个家

也就是在本地硬盘某处，新建一个文件夹，再给这个文件夹起一个自己喜欢的英文名。

这个文件夹未来将存放所有跟这个博客相关的东西，也就是博客的数据库啦。

二、下载 Hexo

  1. 打开新建好的文件夹；
  2. 在空白处点击鼠标右键；
  3. 鼠标左键单击 Git Bash Here，打开 Git 的控制窗口；
  4. 输入 npm install hexo-cli -g，后面等待就好，如果出现报错，无视就好；
  5. 安装结束后，继续输入 hexo -v 验证是否安装成功。

三、初始化博客本地设置

  1. 在 Git 控制窗口输入 hexo init 初始化文件夹；
  2. 输入 npm install 安装必备的组件；

四、生成博客静态网页、预览博客

  1. 在 Git 控制窗口输入 hexo g 生成网页。

    因为 Hexo 是静态博客，所以更新博客内容都要通过这个指令刷新博客。

  2. 输入 hexo s 打开本地服务器，实现本地预览。

    Hexo 本地预览网址为：http://localhost:4000/，把网址输入到浏览器就预览更新后的博客了。

连接 Github 进行托管，将博客互联网化

这是最后的 20% 啦，这部分是为了给博客在互联网安家，也是前置步骤新建 Github 仓库的目的。

  1. 在本地的博客文件夹空白处点击鼠标右键；

  2. 鼠标点击 Git Bash Here 打开 Git 控制窗口 ；

  3. 输入 git config --global user.name "用户名"

    这里的用户名是你的 Github 账户的用户名，后面步骤没有特别说明的都是这个意思；

  4. 输入 git config --global user.email "邮箱地址"

    这里的「邮箱地址」是你绑定 Github 账户的邮箱，后面步骤没有特别说明的都是这个意思；

  5. 输入 ssh-keygen -t rsa -C "邮箱地址"

  6. 打开 Github；

  7. 进入用户设置；

  8. 进入新建 SSH 页面

    然后放置一边；

  9. 在 Git 控制窗口输入 cat ~/.ssh/id_rsa.pub 获得关联用的 key，复制 key；SS

  10. 回到新建 SSH key 的网页，如图所示输入信息；

  11. 在 Git 控制窗口输入 ssh -T git@github.com，确认连接是否成功。

    图上是我的用户名，不要照本宣科啊。

实现与「仓库」同步，真正互联网化

  1. 打开自己 Github 的主页，网址为：https://github.com/自己的用户名；
  2. 进入之前我们创建用来托管博客的「仓库」，看图操作；
  3. 在博客文件夹里找到 _config.yml 文件，双击打开它[1]；
  4. 在打开的文本中找到 deploy 项；
  5. 修改其中的 repository 项的网址；
  6. 在 _config.yml 文本中找到 URL 项；
  7. 将其中的 URL后的网址修改成你的博客静态网址[2];
  8. 使用 Ctrl+S 快捷键保存文档。
  9. 在博客文件夹根目录打开 Git 控制窗口，这个操作现在不用教了吧；
  10. 输入 hexo g，生成静态网页；
  11. 再输入 hexo s，打开本地服务器，去预览生成的网页是不是自己预期的那样；
  12. 最后输入 hexo d，与 Github 仓库同步；
  13. 最后的最后，用浏览器输入自己的博客网址，也就是托管博客「仓库」的静态网址。

一切顺利的话，现在你已经可以看到自己的博客啦，用手机也阅览看看，千辛万苦弄好的成功，是不是很感动。
